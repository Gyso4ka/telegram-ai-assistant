# Telegram AI Assistant

Готовый к использованию бот для Telegram на базе Google Gemini. Поддерживает текстовые диалоги, распознавание изображений, расшифровку голосовых сообщений и персонализированную память.

---

## Функции

- **Текстовый чат** с памятью многоходовых диалогов
- **Анализ изображений** с помощью моделей Gemini Vision
- **Транскрипция голоса** с помощью аудиомоделей Gemini
- **Память пользователя**: предпочтения, факты и история диалогов
- **Абстракция поставщика**, позволяющая позже заменить бэкенд ИИ
- **Ограничение скорости** с опциональной поддержкой Redis
- **Docker-контейнеризированное** развертывание с PostgreSQL и Redis

---

## Технологический стек

- Python 3.12+
- aiogram 3.x
- SQLAlchemy 2.0 async + Alembic
- PostgreSQL (производство), SQLite (тесты)
- Redis (опционально, для хранения конечных автоматов и распределенного ограничения скорости)
- Google Gemini API (бесплатный уровень)

## Настройка

### 1. Клонируйте и создайте виртуальное окружение

```bash
python -m venv .venv
source .venv/bin/activate # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
```

### 2. Настройте переменные окружения

Скопируйте `.env.example` в `.env` и укажите свои реальные учетные данные:

```bash
cp .env.example .env
```

Обязательные значения:

- `BOT_TOKEN` — от [@BotFather](https://t.me/botfather)
- `GEMINI_API_KEY` — из [Google AI Studio](https://aistudio.google.com/app/apikey)

### 3. Запуск миграций

Убедитесь, что PostgreSQL запущен, затем:

```bash
alembic upgrade head
```

### 4. Запуск бота

```bash
python -m app.main
```

---

## Docker Compose (рекомендуется)

Запустите весь стек одной командой:

```bash
cp .env.example .env
# отредактируйте .env, указав ваши реальные BOT_TOKEN и GEMINI_API_KEY
docker compose up --build
```

Это запускает PostgreSQL, Redis и контейнер бота. Контейнер бота автоматически применяет миграции перед запуском.

### Добавление нового поставщика ИИ

1. Реализуйте `AIService` в `services/ai/<provider>.py`.

2. Добавьте ветку в `services/ai/factory.py`.

3. Добавьте настройки, специфичные для поставщика, в `config/settings.py`.

4. Установите `AI_PROVIDER=<provider>` в вашей среде.

## Замечания по безопасности

- Ключи считываются только из переменных среды.

- Длина пользовательского ввода ограничена, удаляются управляющие символы, и проверяется наличие распространенных фраз для внедрения запросов.

- Ограничение скорости помогает защитить поставщика ИИ от злоупотреблений.

- — В логах никогда не содержатся ключи API или секретные ключи сообщений.
